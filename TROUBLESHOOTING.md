# 故障排除指南

本指南提供了常见问题和解决方案。

## 认证相关问题

### 1. 无法登录

**症状：** 登录后被重定向到登录页面

**解决步骤：**
1. 清除浏览器 cookies 和缓存
2. 检查邮箱是否正确
3. 验证 Supabase 凭证是否正确
4. 确保邮箱已验证（如果启用了邮箱确认）

```bash
# 在开发环境中禁用邮箱确认
# 在 Supabase 仪表板：Authentication > Settings > Email Confirmations: Disable
```

### 2. "用户邮箱未确认"错误

**症状：** 收到邮件验证错误

**解决步骤：**
1. 检查垃圾邮件文件夹
2. 在 Supabase 仪表板中手动确认邮箱
3. 或在开发环境中禁用邮箱确认

## 数据库相关问题

### 1. 表不存在

**症状：** `Error: relation "resumes" does not exist`

**解决步骤：**
1. 检查迁移脚本是否运行
2. 在 Supabase SQL Editor 中手动执行 `scripts/001_create_tables.sql`
3. 检查用户是否有权访问表（RLS 策略）

### 2. 权限被拒绝

**症状：** `Error: new row violates row level security policy`

**解决步骤：**
1. 检查用户是否已认证
2. 验证 RLS 策略中的 `auth.uid()` 是否匹配
3. 检查 user_id 是否正确传递
4. 在 Supabase 中查看 RLS 策略：`Authentication > Policies`

### 3. 连接超时

**症状：** `Error: connect ECONNREFUSED`

**解决步骤：**
1. 验证 Supabase URL 是否正确
2. 检查网络连接
3. 在 Supabase 仪表板中验证项目状态
4. 尝试使用 Supabase 提供的连接测试工具

## 文件上传相关问题

### 1. 简历上传失败

**症状：** "上传失败，请重试"

**解决步骤：**
1. 检查文件大小（Blob 存储有大小限制）
2. 验证文件格式是否为 PDF
3. 检查 Blob 存储是否启用
4. 在浏览器控制台检查详细错误信息

```javascript
// 浏览器控制台中
const response = await fetch('/api/upload', { method: 'POST' });
const error = await response.json();
console.log(error);
```

### 2. 无法读取上传的文件

**症状：** 上传成功但无法访问文件

**解决步骤：**
1. 检查文件是否使用私有访问权限上传
2. 验证 `app/api/resume/route.ts` 中的权限设置
3. 确保当前用户有权访问该文件

## AI 功能相关问题

### 1. AI 聊天不工作

**症状：** "生成失败" 或无响应

**解决步骤：**
1. 检查 OpenAI API 密钥是否有效
2. 验证账户是否有足够的 API 额度
3. 检查是否使用了已弃用的模型（使用 gpt-4 而不是 gpt-4-turbo）
4. 在服务器日志中检查错误信息

```bash
# 查看 Vercel 日志
vercel logs
```

### 2. 面试题目生成超时

**症状：** 生成题目请求超时

**解决步骤：**
1. 检查岗位描述是否过长
2. 减少岗位描述的字数
3. 尝试使用更简洁的岗位信息

### 3. 复盘分析结果不准确

**症状：** 分析结果与预期不符

**解决步骤：**
1. 确保面试记录足够详细
2. 检查是否有足够的面试数据（至少 3-5 条记录）
3. 尝试提供更具体的岗位信息

## 性能相关问题

### 1. 应用加载缓慢

**症状：** 首次加载或页面切换很慢

**解决步骤：**
1. 检查网络连接速度
2. 在浏览器开发者工具中检查加载时间
3. 启用浏览器缓存
4. 检查 Vercel 部署是否完成

```bash
# 清除 Next.js 缓存
rm -rf .next
npm run build
```

### 2. API 调用缓慢

**症状：** AI 响应或数据库查询缓慢

**解决步骤：**
1. 检查网络延迟
2. 验证数据库索引是否正确
3. 检查是否有大量数据
4. 在 Supabase 仪表板中查看查询性能

## 界面相关问题

### 1. 样式混乱或不正确

**症状：** CSS 未应用或样式异常

**解决步骤：**
1. 清除浏览器缓存
2. 硬刷浏览器（Ctrl+Shift+R 或 Cmd+Shift+R）
3. 检查 Tailwind CSS 是否正确配置
4. 验证 `globals.css` 是否正确导入

### 2. 表单输入不响应

**症状：** 输入框无法输入或按钮无响应

**解决步骤：**
1. 检查浏览器控制台的 JavaScript 错误
2. 验证表单组件是否正确导入
3. 检查是否有 disabled 属性
4. 尝试重新加载页面

## 部署相关问题

### 1. 构建失败

**症状：** Vercel 构建失败

**解决步骤：**
1. 检查构建日志中的错误信息
2. 验证所有导入路径是否正确
3. 检查 TypeScript 类型错误：`npm run build` 本地测试
4. 确保所有环境变量已设置

### 2. 环境变量未生效

**症状：** 应用无法找到环境变量

**解决步骤：**
1. 重新部署应用
2. 验证环境变量是否在 Vercel 仪表板中正确设置
3. 确保使用了 `NEXT_PUBLIC_` 前缀的变量可在浏览器中访问
4. 检查变量是否包含特殊字符或空格

## 浏览器特定问题

### 1. 仅在特定浏览器中出现问题

**症状：** Chrome 中正常，但在 Safari 或 Firefox 中失败

**解决步骤：**
1. 清除浏览器缓存和 cookies
2. 禁用浏览器扩展
3. 检查浏览器控制台错误
4. 在浏览器兼容性文档中查找已知问题

### 2. 移动设备上的问题

**症状：** 桌面正常，但手机上显示错误

**解决步骤：**
1. 使用浏览器开发者工具的移动视图测试
2. 检查响应式设计是否正确
3. 验证触摸事件处理
4. 检查视口设置
